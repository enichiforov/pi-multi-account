# Maintainers

`pi-multi-account` is maintained as a separate project by the `enichiforov/pi-multi-account` repository owner.

## Maintainer responsibilities

- Keep the extension installable through `pi install git:github.com/enichiforov/pi-multi-account`.
- Preserve compatibility with current pi/GSD runtimes before adding new features.
- Keep internal provider ids stable so existing auth storage, pools, presets, and project restrictions continue to work.
- Require regression coverage for pool selection, project restrictions, subscription labels, presets, and failover behavior.
- Keep upstream attribution and MIT license notices intact while replacing internals over time.
